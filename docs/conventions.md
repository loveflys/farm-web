# 代码规范

## 通用规范

* Soft tab，用两个空格代替tab，不得使用Hard tab
* 缩进单位为两个空格
* 任何代码中不得随意放空行，理论上不得有连续空行
* 任何引入的三方资源或者库，包括不仅限于`npm`, `bower`，都需要在[README](../README.md)中说明

## Javascript规范

* 变量命名：`camelCase`
* 类命名：`PascalCase`

## HTML规范

* `class`和`id`，命名采用全部小写，并且用`-`连接的命名方式

## CSS规范

* 统一采用`scss`来写样式
* 命名规范同`HTML规范`
